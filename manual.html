1. System Overview

The Church Office Bearer Election System is a browser-based voting application designed specifically for conducting Reformed church congregational elections for the offices of Elder and Deacon. It is designed to be run entirely on a single laptop without any internet connection or external server — all data is stored locally in the browser.

How the System Works

The system consists of two HTML files that are opened in a web browser:

FilePurposeWho Uses It
index.htmlAdministration hub — all setup and control functionsElection Officer, Chairman
vote.htmlVoter ballot page — members cast their votes hereAll voting members

Both files share the same data store (browser localStorage). The index.html file is opened on the administrator's laptop. The vote.html page is made available to voters via a local network URL or QR code so they can vote on their own phones or tablets.

Key Features

  • Supports multi-round elections with automatic majority detection
  • Separate elections for the offices of Elder and Deacon
  • Digital voting via personal devices (phones/tablets) using unique token codes
  • Paper ballot entry for members who prefer to vote on paper
  • Password-protected administration and confidential results
  • Real-time participation monitoring
  • Automatic voter page updates — voters are notified when new rounds open
  • No internet required; all data stays on the local device
Important: Because all data is stored in the browser's localStorage, it is tied to the specific browser and device on which index.html is opened. Do not clear browser data or use a different browser during an election.

2. Technical Requirements

Administrator Device (Laptop/PC)

  • Any modern web browser (Chrome, Edge, Firefox, or Safari — version from 2020 or later)
  • The two files index.html and vote.html must be in the same folder
  • JavaScript must be enabled in the browser
  • The device must be connected to a local Wi-Fi network (so voters can reach vote.html)

Voter Devices (Phones/Tablets)

  • Any device with a modern web browser
  • Must be connected to the same local Wi-Fi network as the administrator's laptop
  • No app installation required

Setting Up the Local Network

For voters to access the ballot page on their own devices, the laptop running index.html must serve the files over the local network. The simplest method is to use a local HTTP server. One easy option on Windows is IIS; on Mac, Python can be used:

Open a Terminal or Command Prompt in the folder containing the two HTML files.
Run the command: python3 -m http.server 8080
Find your laptop's local IP address (e.g. 192.168.1.45) using System Preferences > Network (Mac) or ipconfig (Windows).
The voter URL will be: http://192.168.1.45:8080/vote.html
Enter this URL in the Settings tab of Election Setup so it appears on token cards and the QR code.
Tip: Test the URL on a phone before the meeting to confirm connectivity. Voters can also scan the QR code printed on their token card.

3. Passwords & Security

The system uses two separate passwords to protect different areas:

PasswordProtectsDefault
Admin PasswordElection Setup and Round Controlelection2024
Results PasswordElection Dashboard (confidential results)results2024
Action Required: Change both passwords before the election. Default passwords are publicly known. Use the Settings tab in Election Setup to update them.

Passwords are stored as SHA-256 cryptographic hashes — the actual password text is never saved anywhere in the system.

Who Has Which Password

  • Admin password — Election officer only (used to configure and run the election)
  • Results password — Chairman of the meeting only (used to view results on the Election Dashboard)

Keep these passwords separate and confidential. The chairman does not need the admin password, and the election officer does not need to share the results password.

4d. Tokens — Generating & Printing

Click the Tokens tab. Tokens are unique 4-digit codes that identify each eligible voter. Each voter receives one token card which they use for both the Elder and Deacon elections.

Generating Tokens

Enter the number of tokens to generate (equal to the number of eligible voters, e.g. 85).
Click Generate Tokens. The system creates unique 4-digit codes.
Review the list to confirm the correct number was generated.
Screenshot — Tokens Tab
The Tokens tab showing the token generation controls at the top (number input and Generate button), the list of generated token codes displayed in a grid, and print buttons at the bottom right: "Print Token Cards", "Elder ballot quantity + Print Elder Ballots", and "Deacon ballot quantity + Print Deacon Ballots".

Printing Token Cards

Click Print Token Cards. The system formats all tokens as a 4-column grid of cards. Each card includes:

  • The congregation name
  • The 4-digit token code (large, prominent)
  • A QR code linking to the voter page
  • The voting URL in text form

Print on card stock if possible. Cut along the card borders and distribute one card per eligible voter at the door.

Screenshot — Token Cards Print Preview
The print preview showing a 4-column grid of small token cards. Each card contains the church name at top, a large 4-digit code in the centre, a small QR code, and the URL below it. Dashed cut lines separate the cards.
Important: The QR code on token cards comes from the URL configured in the Settings tab. Configure the voter URL in Settings before printing token cards.

Printing Paper Ballots

For members who prefer to vote on paper, the system can print ballot cards. Ballots show all candidates with a checkbox next to each name, plus the number of positions to be filled.

  • Set the desired quantity (e.g. 20) in the Elder ballots quantity field
  • Click Print Elder Ballots — a page with ballot cards in a 2-column grid is printed
  • Repeat for Deacon ballots
  • Cut the ballots and have them available at a paper ballot station
Note: Paper ballots do not include round information or the number of votes per round — this is intentional so that the same printed ballots can be reused across multiple rounds. Verbally inform voters which candidates are valid for each round and how many they may select.
Screenshot — Paper Ballot Print Preview
Print preview showing a 2-column grid of ballot cards. Each ballot card shows the office name (e.g. "OFFICE OF ELDER") at the top, the church name, the number of positions to be filled, then a list of candidate names each with a small square checkbox, and the office name again in the footer.

4e. Settings — URL, QR Code & Passwords

Click the Settings tab. This tab has three sections:

Voter URL

Enter the full URL that voters will use to access the ballot page. Example: http://192.168.1.45:8080/vote.html

Click Save URL. This URL will appear on token cards, the QR code, and the landing page.

QR Code

Once the URL is saved, a QR code is automatically generated and displayed. Voters can scan this code with their phone camera to open the voter page instantly. Click Print QR Code to print a full-page QR code for display in the meeting room.

Change Admin Password

Enter your current password (default: election2024), then a new password (minimum 4 characters), and confirm it. Click Update Admin Password.

Change Results Password

Enter the current results password (default: results2024), a new password, and confirm. Click Update Results Password. Share this new password only with the chairman.

Screenshot — Settings Tab
The Settings tab showing: (1) a Voter URL section with a text input and Save/Clear buttons; (2) a QR Code section with the generated QR code image and a Print QR Code button; (3) a Change Admin Password card with three password fields and an Update button; (4) a Change Results Password card with three password fields and an Update button.

Danger Zone

The Reset All Election Data button at the bottom of the Settings tab permanently erases all nominees, ballots, tokens, and results. Two confirmation prompts prevent accidental use. Only use this to start fresh before a new election.

5. During the Meeting — Round Control

Round Control is used during the meeting to manage all aspects of the voting process. It is the central control panel for the election officer.

5a. Accessing Round Control

From the landing page of index.html, click Round Control.
Enter the admin password and click Enter Round Control.
Screenshot — Round Control Login
The login screen with subtitle "Enter the admin password to access Round Control" and button labelled "Enter Round Control".

5b. Starting Round 1

When you first enter Round Control after completing setup, the system detects the configured Elder election and shows a "Ready to Start" panel:

Screenshot — Round Control Ready to Start (Elder)
The Round Control header is shown in dark brown (Elder colour). A panel displays "Elder Election — Configured" with the number of nominees, positions to fill, and votes per voter. Below is a large "Start Round 1 Voting" button.

The panel confirms:

  • Number of nominees configured
  • Number of positions to fill
  • Votes per voter in Round 1

Click Start Round 1 Voting to begin. This immediately opens voting and makes the ballot available to all voters.

Tip: Do not click Start Round 1 until the chairman has introduced the candidates and is ready for voting to begin.

5c. Opening and Closing Voting

Once a round is active, the Round Control screen shows the control bar:

The Round Control screen showing: a coloured header with office name and round number; a sequence bar (Elder → Deacon → Complete) showing current progress; four stat boxes (Ballots In, Paper Ballots, Expected Voters, Participation %); a control bar with a green pulsing "Voting Open" indicator, greyed-out "Open Voting" button, active "Close Voting" button, and greyed-out "End Round" button; and a results section below showing candidate names with vote bars and percentages.

5d. Monitoring Participation

The four statistics boxes update automatically every 3 seconds:

StatMeaning
Ballots InTotal number of ballots received (digital + paper) for the current round
Paper BallotsNumber of ballots entered via the Paper Ballot Entry station
Expected VotersThe number entered in Election Setup. Used as the denominator for percentage.
Participation %Ballots In ÷ Expected Voters × 100. Shown in green when ≥ 50%.
Note: The vote counts per candidate visible in Round Control are for the election officer's operational use only. These are not shared publicly. The confidential results are only available on the Election Dashboard.

5e. Ending a Round & Round Transition

Once voting is closed, click End Round. The Round Transition screen opens.

Screenshot — Round Transition Screen
The Round Transition screen showing: a coloured header; a "Final Results — Round X" section listing all candidates ranked by position (numbered circles); Step 1 with checkboxes to mark elected candidates (with a yellow "majority" badge where applicable); Step 2 with checkboxes for candidates to advance to the next round; Step 3 showing the next ballot candidate list with Remove/Restore buttons; Step 4 with the votes-per-voter field for the next round; and at the bottom, either a "Launch Round X+1" button or "Complete This Office" button.

The Round Transition screen has four steps:

5f. Confirming Elected Candidates (Step 1)

The system automatically detects candidates who received a majority of votes (more than 50% + 1 of all ballots cast) and pre-checks them with a majority badge.

For each candidate in the list:

  • Check the box to mark them as elected in this round
  • Uncheck to override the auto-suggestion if the congregation disputes the result
Majority Rule: A candidate must receive strictly more than half the total ballots cast (floor(n/2) + 1 or more votes) to be automatically suggested as elected.

5g. Configuring the Next Round (Steps 2–4)

Step 2 — Candidates Advancing: All non-elected candidates are automatically listed to advance to the next round. Uncheck any candidate you wish to remove from the next round.

Step 3 — Next Ballot Preview: Shows which candidates will appear on the next round's ballot. You can remove individual candidates using the Remove button (click Restore to undo).

Step 4 — Votes Per Voter: Set how many candidates each voter may select in the next round. The system auto-fills this as the number of remaining positions to fill. Adjust if needed.

Screenshot — Round Transition Steps 2–4
Steps 2, 3, and 4 of the Round Transition screen. Step 2 shows checkboxes for non-elected candidates with "advanced" tags. Step 3 shows the resulting next ballot list with Remove buttons. Step 4 shows a number input for votes per voter with an auto-fill hint showing "X positions total · Y elected so far · Z remaining — auto-filled". At the bottom is a "Launch Round 2 Voting" button.

5h. Completing an Office

When all positions for an office have been filled (all elected candidates meet or exceed the number of positions), the system detects this and shows a Complete This Office button instead of a next-round button.

Screenshot — All Positions Filled Banner
A gold banner reading "All X positions filled — Y candidate(s) elected" appears above the Complete This Office button. The "Launch Next Round" section is hidden.

Click Complete This Office. The system:

  1. Records the final election results for that office
  2. Automatically moves to Round Control for the next office (Deacon) showing the "Ready to Start" panel

Repeat the entire Round Control process for the Deacon election.

When the Deacon election is also complete, clicking Complete This Office will take you back to the landing page.

6. Paper Ballot Entry

The Paper Ballot Entry screen is used by a dedicated volunteer to enter votes on behalf of members who voted on paper. It is accessible from the landing page without a password.

Important: Only open Paper Ballot Entry on the administrator's laptop, not on any voter device. Ensure voting is open before entering paper ballots.

6a. Entering a Paper Ballot

Screenshot — Paper Ballot Entry Screen
The Paper Ballot Entry screen showing: the current office and round number at the top; a list of candidate names as selectable rows (each with a checkbox icon); an optional Member ID field; a "Submit Paper Ballot" button; and a log of previously entered paper ballots at the bottom.
The current office and round are shown automatically. If voting is closed, a warning message is displayed.
Click on each candidate's name that appears on the paper ballot to select them. Selected candidates are highlighted. The number of allowed selections is enforced.
Optionally enter a member identifier in the Member ID field (e.g. a ballot number) for tracking purposes.
Click Submit Paper Ballot. The vote is recorded and the form resets.
The submitted ballot appears in the log at the bottom of the screen.

6b. Editing a Submitted Ballot

If a mistake is made, any paper ballot from the current round can be corrected:

Find the ballot in the log at the bottom of the screen.
Click the Edit button next to the ballot entry.
The form re-populates with the previous selections. An orange "Editing Ballot #X" banner appears.
Make the necessary corrections by clicking candidates to deselect/reselect.
Click Update Paper Ballot to save the correction, or Cancel to discard changes.
Screenshot — Paper Ballot Edit Mode
The Paper Ballot Entry screen in edit mode, showing an orange "Editing Ballot #3" banner above the candidate list, the previously selected candidates already highlighted, an "Update Paper Ballot" primary button, and a "Cancel" secondary button.

6c. Printing Paper Ballots

Paper ballots are printed from the Tokens tab in Election Setup (before the meeting). See Section 4d for details. Each ballot card shows:

  • Office name (Office of Elder or Office of Deacon)
  • Congregation name
  • Number of positions to be filled
  • All nominees with a checkbox next to each name

Because round information is not printed on the ballots, the same printed ballots can be reused across multiple rounds. The chairman verbally announces which candidates are valid for each round and how many votes each member may cast.

7. The Voter Page

The voter page (vote.html) is what voting members see on their phones or tablets. It automatically updates every few seconds to reflect the current state of the election.

7a. Accessing the Voter Page

Members can access the voter page in two ways:

  • QR Code: Scan the QR code on their token card with their phone camera
  • URL: Type the URL printed on the token card into their phone's browser

7b. Entering a Token

When voting is open, the voter page shows the Token Entry screen:

Screenshot — Voter Page Token Entry
The voter page showing a white card on a dark navy background. The card header shows the congregation name, meeting date, and a green "Voting Open" pill. Below is a large "Enter Your Voting Token" heading, instruction text, a large 4-digit numeric input field, and a "Submit Token" button.
The voter opens the voter page URL or scans the QR code on their token card.
They type their 4-digit token code into the input field.
Tap Submit Token.

If the token is invalid or has already been used for the current round, an error message is shown and the voter must try again.

7c. Casting a Vote

After submitting a valid token, the ballot is displayed:

Screenshot — Voter Ballot Screen
The ballot screen showing the office name badge (e.g. "ELDER · ROUND 1"), a heading "Cast Your Vote", instruction text showing how many candidates to select, a list of candidate names as tappable rows (each with a circle checkbox that fills when selected), a vote counter showing how many selections have been made, and a "Submit Vote" button that becomes active when the correct number of candidates are selected.
The voter reads the list of candidates. The number of candidates they may select is shown.
Tap each candidate's name to select them. A checkmark appears next to selected candidates.
The voter may select up to the permitted number of candidates but is not required to use all selections.
Tap Submit Vote when satisfied. A confirmation screen appears.
Votes are final once submitted. There is no way for the voter to change their vote after submitting. If a member makes an error, they must inform the election officer who can make corrections only via the Paper Ballot Entry screen (for paper ballots) — digital votes cannot be edited after submission.

From the landing page of index.html, click Election Dashboard.

9. Complete Election Walkthrough

This section walks through a complete election from start to finish using a realistic example: electing 2 Elders and 1 Deacon from a congregation of 70 voters.

Example Setup

ElderDeacon
NomineesJohn Smith, Peter Johnson, Michael Williams, David BrownAndrew Taylor, James Wilson, Robert Davis
Positions21
Votes per Voter (Rd 1)21
Expected Voters70

Phase 1 — Before the Meeting

Open index.html in Chrome on the meeting laptop. You see the landing page.
Go to Election Setup → enter admin password → Settings tab → enter the voter URL (e.g. http://192.168.1.10:8080/vote.html) → Save URL. The QR code updates.
Change passwords. In Settings, update the Admin Password and the Results Password. Write them down securely.
Go to Details tab. Enter congregation name ("Grace Reformed Church"), meeting date, and 70 as expected voters. Tab away to auto-save.
Go to Elder tab. Enter the four nominee names (one per line), set Positions to 2, Votes per Voter to 2. Click Save Elder Setup.
Go to Deacon tab. Enter the three nominee names, Positions = 1, Votes per Voter = 1. Click Save Deacon Setup.
Go to Tokens tab. Enter 70 and click Generate Tokens. Click Print Token Cards. Print and cut 70 token cards.
Print paper ballots. Set Elder ballots qty to 15, click Print Elder Ballots. Repeat for Deacon (qty 15). These are for members who prefer paper.
Test connectivity. Open the voter URL on a phone, confirm the token entry screen appears.
Distribute token cards to eligible voters as they arrive at the meeting.

Phase 2 — Elder Election, Round 1

At the appropriate point in the agenda, the chairman announces the start of Elder voting.
Election officer goes to Round Control → enters admin password → clicks Start Round 1 Voting. Voting is now open.
Voters open the voter URL, enter their token, and cast their votes — selecting up to 2 candidates.
Paper ballot volunteers distribute paper ballots, collect them, and enter them via the Paper Ballot Entry screen.
The election officer monitors the Participation % in Round Control. When the chairman is satisfied, the election officer clicks Close Voting.
Click End Round. The Round Transition screen opens.

Example results after Round 1:

CandidateVotesMajority (>35)?
John Smith42Yes ✓
Peter Johnson38Yes ✓
Michael Williams22No
David Brown18No
Both John Smith and Peter Johnson are auto-suggested as elected (majority badges). Their checkboxes are pre-checked. Confirm by leaving them checked.
Since 2 positions are filled with 2 majority candidates, the system shows the Complete This Office button.
Click Complete This Office. The Elder election is done.

Phase 3 — Deacon Election, Round 1

After clicking Complete This Office, the system automatically shows Round Control for the Deacon election in "Ready to Start" mode.
The chairman announces the start of Deacon voting. Voters on the "Voting Round Closed" screen will automatically advance to the token entry screen within 3 seconds when voting opens.
Election officer clicks Start Round 1 Voting.
Voting proceeds. When done, click Close Voting → End Round.

Example results after Deacon Round 1:

CandidateVotesMajority (>35)?
Andrew Taylor28No
James Wilson25No
Robert Davis17No

No candidate received a majority. A second round is needed.

Phase 4 — Deacon Election, Round 2

In the Round Transition screen, no candidate is auto-suggested as elected. Step 1 checkboxes are all unchecked.
Step 2 shows all three candidates checked to advance. The chairman may decide to remove the lowest-ranked candidate. Uncheck Robert Davis in Step 2.
Step 3 shows the next ballot: Andrew Taylor and James Wilson. Confirm 1 vote per voter (auto-filled).
Click Launch Round 2 Voting. Voters who are on the "Vote Submitted" screen see a notification: "Round 2 is now open — Vote Now →"
After Round 2 voting: Andrew Taylor receives 40 votes (majority). He is auto-suggested as elected.
Confirm Andrew Taylor as elected in Step 1. Click Complete This Office.

Phase 5 — Election Conclusion

After completing the Deacon election, the system returns to the landing page.
Voters' screens automatically change to "Election Complete — Thank you for participating."
The chairman opens the Election Dashboard with the results password and announces the elected officers to the congregation.

10. Troubleshooting

ProblemLikely CauseSolution
Voters cannot reach the voter page Devices not on the same Wi-Fi network, or the URL is wrong Confirm all devices are on the same Wi-Fi. Check the URL in Settings. Try typing the URL manually.
"Invalid token" error when voter submits Token code mistyped, or the token list was regenerated after cards were printed Double-check the 4-digit code. If tokens were regenerated, print and redistribute new cards.
"This token has already been used" error The voter already voted in this round, or someone else used their token Check the Paper Ballot log. If a duplicate entry was made by mistake, edit it. Contact the voter to verify.
Expected Voters shows 100 in Round Control The Expected Voters field was not saved in Election Setup Go to Election Setup → Details tab → correct the value and tab away (auto-saves) or click Save Details.
Voter page shows "No Active Election" No office has been configured, or all offices are marked complete Check that nominees are saved in Election Setup for both offices.
Voter page stuck on "Voting Not Yet Open" Round 1 has not been started in Round Control Go to Round Control and click Start Round 1 Voting. The voter page updates automatically within 3 seconds.
Percentage stays at 0% in Round Control Expected Voters is set to 0 Update Expected Voters in Election Setup Details tab to the correct number.
Forgotten admin password Password was changed and forgotten There is no password recovery. You must Reset All Election Data in Settings (loses all data) and start over. Keep the password written down in a safe place.
After completing Elder, Deacon does not show Deacon nominees were not saved in Election Setup before the meeting Go to Election Setup → Deacon tab → add nominees and click Save Deacon Setup. Then go to Round Control.
All data was lost (browser cleared/different browser) localStorage was cleared, or a different browser/profile was used All data is stored in the specific browser profile that was used. Always use the same browser and do not clear browsing data. Consider exporting results from the Dashboard before closing the browser.
